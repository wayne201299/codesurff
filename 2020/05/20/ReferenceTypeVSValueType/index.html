Reference type VS Value type

Posted on 2020-05-20

Value Type (實值型別)

  • 實際的值，整數、布林…，是直接傳value，在記憶體空間內儲存的是實際的value，代表取該變數的話可以直接取到他的值
  • 傳值的方式如下圖，直接把變數內儲存的值傳遞出去，pass by value
    Example :

value type的資料型別

  • bool
  • byte
  • char
  • decimal
  • double
  • enum
  • float
  • int
  • long
  • sbyte
  • short
  • struct
  • uint
  • ulong
  • ushort

    Reference Type (參考型別)

  1. 不同於value type，reference type儲存的是記憶體位置，甚麼意思呢?在了解這部分前要先知道stack 跟heap
    1. stack 用於靜態記憶體配置
    2. heap 用於動態記憶體配置
  2. 用下面這張圖來解釋，value type的變數和他的值會存在stack中，然而reference type的stack中實際存的值是記憶體位置，那value則存在於heap中，所以被指定到同一個reference type的變數其實是指向同一個記憶體位置，隨著該記憶體位置的值改變，pass by reference

    Example :

reference type的資料型別

  • String
  • All arrays, even if their elements are value types
  • Class
  • Delegates

使用時機

  • 看有沒有限制使用的資料型別，例如傳遞的變數是class或integer

使用value type

  • 需要考慮資料是不是會被多個其他執行序的程式使用到，而你不希望資料在這過程中被改動
  • 數值比較中用到==，這是單純用來比較值，而非位址

使用reference type

  • 希望這個變數是共享的，也就是資料能在其他function中修改
  • 用===做比較，這是用來表示絕對相等，包含記憶體位址
